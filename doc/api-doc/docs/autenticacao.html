Autenticação

A API da Hunter Wallet utiliza um mecanismo de autenticação challenge-response baseado em JWT e HMAC para garantir a segurança das requisições.

Para fins de desenvolvimento, você pode usar o gerador de credenciais para obter um DeviceID e AppSignature válidos.

Fluxo de Autenticação

O fluxo de autenticação segue estes passos:

  1. Obtenha um desafio (challenge) do servidor
  2. Assine o desafio com sua chave secreta usando HMAC-SHA256
  3. Envie a assinatura de volta para o servidor
  4. Receba um token JWT para autenticação nas requisições subsequentes

1. Solicitar Desafio

Envie uma requisição GET para obter um desafio único do servidor.

GET /v1/public/challenge

Cabeçalhos

Nome Tipo Obrigatório Descrição
X-Device-ID String Sim Identificador único do dispositivo

Resposta de Sucesso (200 OK)

{
    "status": "success",
    "statusCode": 200,
    "data": {
        "challenge": "a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8",
        "expiresIn": 300
    }
}

2. Gerar Assinatura

Use o valor do desafio recebido para gerar uma assinatura HMAC-SHA256.

Exemplo em JavaScript 
const crypto = require('crypto');

function generateSignature(challenge, secretKey) {
    return crypto
        .createHmac('sha256', secretKey)
        .update(challenge)
        .digest('hex');
}

const challenge = 'a1b2c3d4-e5f6-7890-g1h2-i3j4k5l6m7n8';
const secretKey = 'sua-chave-secreta-aqui';
const signature = generateSignature(challenge, secretKey);

console.log('Assinatura gerada:', signature);

3. Autenticar com a Assinatura

Envie a assinatura gerada para autenticar e receber o token JWT.

POST /v1/public/authenticate

Cabeçalhos

Nome Tipo Obrigatório Descrição
X-Device-ID String Sim Mesmo Device-ID usado na solicitação do desafio
X-Signature String Sim Assinatura HMAC-SHA256 do desafio

Corpo da Requisição

Nenhum corpo necessário para esta requisição.

Resposta de Sucesso (200 OK)

{
    "status": "success",
    "statusCode": 200,
    "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "expiresIn": 3600,
        "tokenType": "Bearer"
    }
}

Usando o Token JWT

Após obter o token JWT, inclua-o no cabeçalho Authorization de todas as requisições autenticadas:

Exemplo de Uso do Token 
// Exemplo de requisição autenticada
const response = await fetch('https://api.hunterwallet.com/v1/me', {
    headers: {
        'Authorization': 'Bearer seu-token-jwt-aqui'
    }
});

Mantenha seu token JWT seguro e nunca o expon em código do lado do cliente. O token deve ser armazenado de forma segura no lado do cliente.

Erros Comuns

400 Bad Request

Ocorre quando os parâmetros fornecidos são inválidos ou ausentes.

401 Unauthorized

Ocorre quando a autenticação falha ou o token JWT é inválido/expirado.

403 Forbidden

Ocorre quando o usuário não tem permissão para acessar o recurso solicitado.

429 Too Many Requests

Ocorre quando o limite de requisições foi excedido. Tente novamente mais tarde.
